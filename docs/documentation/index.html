Documentation

Bot Basics

There are some general rules you should be aware of

  • Bots move forward one space when moving
  • Dodgeballs move forward two spaces each turn
  • All bot moves are received by the server and then the moves executed. For this reason, it is totally possible for two bots to kill each other on the same turn
  • If a bot stays on a single space for more than 120 turns, it will be killed due to a timebomb mechanic. Rotating does not reset this, only moving off the square will reset the timer.
  • A throw dodgeball will fall to the ground whenever it hits a player, a wall, a moving dodgeball or a fallen dodgeball.
  • The most recent game is always saved in the lastgame.pickle file. If you’d like to save a game replay for later analysis, just rename this file! Otherwise it will be rewritten each game.

Available Commands

There are currently 5 commands which a bot can execute:

Command Description
‘forward’ Takes one step forward in the currently facing direction. If there is a wall in front of the bot, nothing happens
‘rotCW’ Rotates the bot 90 degrees in a clockwise direction
‘rotCCW’ Rotates the bot 90 degrees in a counter-clockwise direction
‘ball’ Assuming the bot still have dodgeballs in its inventory, throws a dodgeball in the bot’s currently facing direction
‘ping’ The bot stays stationary for a turn to gather information about its surroundings in an 8 block radius. More information in the vision section below.

Making Informed Choices

Attributes

Each turn the server returns a variety of potentially useful information to the bot. Information is stored in the bot class and accessed via atributes. For example:

bot.playercount

would get the number of players still alive in the game. Currently available attributes are:

Attribute Description
.active Returns true if the bot is still alive and the game is still ongoing
.playercount Returns the current number of bots still in the game
.ballcount Returns the number of dodgeballs the bot currently has available
.vision Returns a list of number values relating to what your bot “sees” directly ahead of it. See vision below for more information.
.lastping Returns a dictionary of all the last pinged information. See vision below for more information.

Vision

One of the most important things you’ll likely want to consider when deciding what move your bot should make each turn is what is currently sees in front of it. The .vision attribute will return a list of numbers that correspond to what your bot sees in front of it. This list starts with the bot itself and proceeds until a wall is encountered. Entries in the list follow the following conventions:

Number Meaning
0 There is nothing here. Empty space.
1 There is a wall here. Since vision ends at a wall (and doesn’t include the wall), you shouldn’t see this number come up.
2 This is a active and moving dodgeball, and is therefore dangerous! Moving dodgeball will also have a decimal value, which indicates what direction they are facing (and thus moving).
3 This is an inactive and fallen dodgeball. Moving over it will pick it up and add it to your number of available dodgeballs.
2-digit # All bots are identified with unique 2-digit numbers, so this indicates that an enemy bot is located here. Bots will also have a trailing decimal value that indicates what direction they are currently facing.

Facing direction is given by the decimal value:

  • 0.0 - Facing North
  • 0.1 - Facing East
  • 0.2 - Facing South
  • 0.3 - Facing West

Ping Vision

If your bot has performed a ‘ping’ move, it will have data in the .lastping attribute that it can access. Unlike normal vision, the ping vision is given in the form of a dictionary. Everything seen by the ping is given by a dictionary key:

Key Description
‘Terrain’ Any walls or obstacles in the bot’s ping range
‘ABall’ Any active and moving dodgeballs in the ping range. Does not give facing direction!
‘DBall’ Any inactive or fallen dodgeballs in the ping range.
‘Enemy’ Any enemy bots in the ping range. Does not give facing direction!

Each key will return a list of coordinates, where each coordinate corresponds to one instance of the described object. Coordinates are given in the form (x,y) and are relative to the bot itself. At the moment, the directionality of the North-South direction is also flipped, so negative y-coordinates indicate something is above you.

Example: You run bot.lastping['ABall'] and it returns:

[(-5,1), (3,-5)]

this would indicate that there is an active dodgeball 5 squares to the West and 1 square South, as well as another active dodgeball 3 squares to the East and 5 squares North.

Ping will return all objects of interest in an 8-block radius from your current position!

Running the Server

The server is run from a command prompt or shell following normal python conventions. Bots to compete are added to the prompt following a -i option. For example, to run a competition between the RandomMan.py and SimpleMan.py, you’d write:

python server.py -i RandomMan.py SimpleMan.py

There are several other server flags that may be of use:

Flag Default Use
-i Name of bots desired to be in game. Don’t include the Bots/ folder that they reside in.
-d num 0 Playback multiplier to be passed to the viewer. Values of 1-5 will speed up, fractional values will slow down
-s num 20 Sets the square size of the arena
-o num 10 Sets the maximum number of obstacles scattered about the map
-v Using this flag will suppress the viewer from auto-playing after the battle is over
–noprint Using this flag will suppress all printed output to the screen

Running the server with the -h option will output a help file showing these same flags.

Running the Viewer

The viewer script will by default load the last game replay and play it using a matplotlib interface. It is called automatically by default, but can also be run anytime standalone to rewatch a match. It has a few options:

Flag Default Use
-i lastgame.pickle If you’ve saved a game replay under a different name, you can load it with the -i flag
-d 1 Speed multiplier of default playback. Matplotlib seems to max out on a multiplier of >5. Fractional values will give slower playback

Again, running the viewer with the -h option will output a help file showing the above flags if you forget.
