Getting Started

Installation

As Digital Dodgeball is a game based on the Python programming language, you are going to need a working version of Python 3.

  • For Windows or Mac users, the easiest method is likely installing the Anaconda Distribution, which can be found here. This will get you all the needed packages and modules as well as the Spyder IDE.
  • For Linux users, you should just be able to install Python 3 through your distribution’s package manager. As they may be packaged separately, you’ll also want Numpy and Matplotlib.

Getting the Game Files

Once you have Python on your system, you need to grab the game files. You can grab these either directly from my GitHub page or as a zip file.

  • If you are GitHub savy, you can just open a command line and run

    git clone https://github.com/jrembold/WUPhys-CodeComp.git

    which will create a game directory in whatever folder you were in.

  • If you are not yet on the GitHub fan-train, you can grab the identical files in the form of a zip file, which you can then unzip wherever you might like. You can grab the files here.

Writing a basic bot

With the files on your system, you are all set to start constructing your first bot! First understand that a “game” of Digital Dodgeball involves two separate programs running: the individual code for your bot and the server code that actually runs the game. As such, you will never run your bot’s code directly! You will always write or edit the bot code, same it, and then run the separate server script to actually run the game. This is counter to most programs where you write and run them directly, so don’t get this confused!

To walk you through the basics of bot creation, I’ll step you through the basics of RandomMan.py. There are 5 example bots included in the Bots folder which are heavily commented to describe what is happening. Check those out if you have questions or are curious about different ways of implementing the code. For now, open up a text editor (or Spyder) and follow along with the creation of RandomMan.py!

RandomMan.py starts with 3 lines that you always must have in order for your bot to run properly:

import sys
sys.path.append('./')
import library as lib

The first two lines import the sys module for the sole purpose of running the second line. This is necessary so that we can keep the bots in a separate folder from the main directory. The 3rd line imports the core library files that your bot needs to understand to know how to talk to the game server and process moves.

In addition, we are creating a bot that will move randomly here, so we’ll go ahead and import the random module as well

import random

Here you could import any other common module that you think would help you write your script. Numpy is a common module I’ll frequently load for other bots. As long as it is decently common, my server shouldn’t have an issue running it when you submit the code to me.

The next line is to initialize the bot. This activates all the various class functions that you’ll want to use and registers the bot with the server to let it know about the new player! This will always take the form:

bot = lib.CBot(__file__)

Some of the naming conventions you could change if you wanted and understand what you are doing, but the CBot(__file__) portion should stay unchanged. CBot is the needed class from the library module and __file__ tells the server that your bot’s name is the same as whatever you called the filename. Otherwise, the server won’t know what script to associate with your wins!

Now that the server knows of your existence, you should pause to get the latest information from the server. To do this, you can use the:

bot.getMapState()

command. This will cause your bot code to wait until it hears back from the server about how the game is progressing. You’ll want to run it each turn. Then, as long as the server hasn’t told us we are dead (which can’t happen the first turn!), we want to issue commands to our bot! For RandomMan.py, he just chooses 1 of the 5 possible moves randomly each turn. Put together, this will look like

while bot.active:
    move = random.choice(['forward', 'rotCW', 'rotCCW', 'ball', 'ping'])
    bot.sendMessage( move )
    bot.getMapState()

We’ll break this down row by row. The first just ensures that your bot is still almong the living before sending a message back to the server. The server doesn’t take kindly to dead bots trying to play the game. Then we choose what move we’d like to make. Digital Dodgeball has 5 moves available at the moment, and each turn you can submit 1 move to the server. Here, we just randomly choose one from almong the 5. Then we send that move along to the server with bot.sendMessage. Finally, we go ahead and pause to get the map state again. This will let us know the results of all the moves that other bots sent in that turn and give us information to make our next move choice.

And that is it! All together, it looks something like:

# -- Imports --
import sys              #importing sys for the below command
sys.path.append('./')   #this lets us keep the bots in a separate directory
import library as lib   #importing the main library functions
import random           #This bot needs the random library

# -- Initialize Bot --
bot = lib.CBot(__file__)  #this initializes our bot and lets the server know what script ran it (the __file__ part)

# -- Get the first map state --
bot.getMapState()

# -- Main Code --
while bot.active:           #for as long as the bot is alive, do the following
    # Now we choose randomly from a list of our available moves
    move = random.choice(['forward', 'rotCW', 'rotCCW', 'ball', 'ping'])

    # And then we send the move to server
    bot.sendMessage( move )

    # Then get the map state once more
    bot.getMapState()

I’ve included a BotTemplate.py file in the main directory that you can use for a starting point for creating new bots.

Running the Server

At the moment the server can only be run from a command prompt. If you are comfortable with a command prompt, you just need to open a terminal and then navigate your way to the game directory using cd. In windows, you can simplify your life being holding Control and then right-clicking in the game directory. In the menu that pops up, one of the options should be something like “Open command prompt here”. Clicking this will open a terminal already navigated to the correct folder. There may be a similar command with Mac’s, but I don’t know it so someone feel free to enlighten me!

The server is another python script, so you can run it via

python server.py -i "RandomMan.py" "RandomMan.py"

The -i flag always precedes a list of which bots you want to compete. You can think of it as standing for which bots are In the game. Here we are just running two copies of RandomMan.py against one another. You can add as many different or duplicate bots as you like, though if the map is too small the server won’t be able to place them all far enough from each other at the start. After the match concludes, a matplotlib window should pop up which shows you a graphical depiction of what happened in the game. There are some other option parameters you can pass to the server, which are detailed in the Documentation link to the left.

Those are the basics! If you get hung up on anything, be sure to check out the documentation, or come talk to me in Collins 311!
